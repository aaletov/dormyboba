# Функциональные требования
---
| Конфигуратор | Требования |
|--------------|------------|
| NF_Platform | Приложение разрабатывается для платформы VK |
| NF\_ProgrammingLanguage | Разработка должна вестись на языке программирования Python |
| NF\_PythonVersion | Минимальная поддерживаемая версия Python 3.10 |
| NF\_Language | Язык интерфейса бота - русский |
| | |
| F\_Bot_1 | При нажатии на кнопку "Информация о боте" выводится общая информация о боте |
| | |
| F\_AddUserInvite_1 | При нажатии на кнопку "Пригласить нового пользователя" выдаётся ссылка или QR-код для приглашения нового пользователя |
| F\_AddUserInvite_2 | При переходе по приглашению пользователь должен быть перенаправлен на страницу авторизации приложения во "ВКонтакте" |
| | |
| F\_AddUserRegister_1 | После авторизации приложения пользователю должно быть отправлено сообщение с приветствием и просьбой закончить регистрацию |
| F\_AddUserRegister_2 | В ходе диалога регистрации нужно получить имя / псевдоним пользователя и его группу |
| | |
| F\_MailingCreateMailing_1 | При нажатии на кнопку "Создать рассылку" у администратора есть возможность: задать фильтры (FilterButton), время рассылки (TimePrompt), тело сообщения (TextPrompt), подтвердить рассылку (Done), вернуться на шаг назад (Back) |
| F\_MailingCreateMailing_2 | При нажатии на кнопку "Задать фильтры " у администратора есть возможность выбрать фильтры по: курсу (SetCourse), группе (SetGroup), институту студента (SetInstitute), либо отправить рассылку без фильтров (по умолчанию - всем) |
| F\_MailingCreateMailing_3 | У администратора не должно быть возможности задать фильтры, содержащие несуществующую группу пользователей |
| F\_MailingCreateMailing_4 | При нажатии на кнопку "Задать время" у администратора есть возможность отправить сообщение сейчас (по умолчанию), или выбрать дату и время отправки сообщения |
| F\_MailingCreateMailing_5 | У пользователя не должно быть задать время отправки меньшее текущего времени |
| F\_MailingCreateMailing_6 | При нажатии на кнопку "Задать сообщениe" у администратора есть возможность задать тело сообщения |
| F\_MailingCreateMailing_7 | Максимальная длина сообщения соответствует максимальной длине сообщения, указанной в версии VKAPI, используемой в текущей версии продукта |
| F\_MailingCreateMailing_8 | При нажатии на кнопку "Готово" администратор необходимо вывести диалог подтведжения создания рассылки |
| F\_MailingCreateMailing_9 | При нажатии на кнопку "Подтвердить" у администратора должна быть возможность подтвердить создание рассылки или вернуться к её редактированию |
| F\_MailingCreateMailing_10 | На каждом этапе создания рассылки у администратора должна быть возможность вернуться на предыдущий шаг процесса |
| | |
| F\_MailingSendMailing_1 | Сообщение рассылки должно быть отправлено пользователям в виде сообщения в диалоге бота |
| F\_MailingSendMailing_2 | Сообщение должно быть отправлено только пользователям, соответствующим фильтрам, заданным на этапе создания рассылки |
| F\_MailingSendMailing_3 | При отсутствии указанного времени рассылки, сообщение должно быть отправлено сразу после подтверджения создания рассылки |
| F\_MailingSendMailing_4 | При наличии указанного времени рассылки, сообщение должно быть отправлено в указанное время рассылки |
| | |
| F\_QueueCreateQueue_1 | При нажатии на кнопку "Создать новую очередь" у администратора есть возможность: задать название очереди (QueueName), задать описание очереди (QueueInfo), задать время открытия доступа (QueueAccessTime), задать время исчезновения (QueueExpireTime) |
| F\_QueueCreateQueue_2 | При нажатии на кнопку "Задать название очереди" у администратора должна быть возможность задать название очереди |
| F\_QueueCreateQueue_3 | При нажатии на кнопку "Задать описание очереди" у администратора должна быть возможность задать описание очереди |
| F\_QueueCreateQueue_4 | При нажатии на кнопку "Задать время открытия доступа" у администратора должна быть возможность указать время открытия доступа к очереди |
| F\_QueueCreateQueue_5 | При нажатии на кнопку "Задать время закрытия доступа" у администратора должна быть возможность указать время закрытия доступа к очереди |
| | |
| F\_QueueUserInterfaction_1 | При открытии доступа к очереди все студенты получают сообщение о том, что очередь была открыта |
| F\_QueueUserInterfaction_2 | После открытия доступа к очереди у пользователя должна быть возможность записаться в очередь |
| F\_QueueUserInterfaction_3 | При нажатии на кнопку "Войти в очередь" студент может войти в конец очереди |
| F\_QueueUserInterfaction_4 | При нажатии на кнопку "Выйти из очереди" студент может выйти из очереди |
| F\_QueueUserInterfaction_5 | При передаче студенту очереди, студент должен получить уведомление о том, что ему была передана очередь |
| F\_QueueUserInterfaction_6 | При нажатии на кнопку "Закончить" очередь передаётся следующему пользователю |
| | |
| NF_JournalGoogleSheets | В качестве платформы для журнала проблем должно использоваться приложение Google Sheets |
| | |
| F\_DefectCreate_1 | У пользователя должна быть возможность создать запись в журнале проблем |
| F\_DefectCreate_2 | При нажатии на кнопку "Сообщить о проблеме" должен начинаться диалог создания записи в журнале проблем |
| F\_DefectCreate_3 | В диалоге создания записи в журнале проблем у пользователя должна быть возможность выбрать категорию проблемы из вариантов "Сантехника" / "Электрика" / "Общее" |
| F\_DefectCreate_4 | В диалоге создания записи в журнале проблем у пользователя должна быть возможность задать описание проблемы |
| F\_DefectCreate_5 | Описание проблемы ограничено 512 символами |
| F\_DefectCreate_6 | При нажатии кнопки "Готово" запись должна быть сохранена и добавлена в журнал проблем |
| | |
| F\_DefectStatus_1 | После создания проблемы статус проблемы должен быть - "Добавлено" |
| F\_DefectStatus_2 | У администратора должна быть возможность изменить статус проблемы на один из статусов "Принято" / "Решено" |
| F\_DefectStatus_3 | При изменении статуса проблемы пользователю должно быть отправлено сообщение об изменении статуса проблемы |
